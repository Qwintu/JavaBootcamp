# Часть I
### File Signatures

|||  
|--|--|  
| **Разрешенные типы** | Java Collections API (`List<T>`, `Map<K, V>`, etc.), InputStream, OutputStream, FileInputStream, FileOutputStream |


В Java классы ввода/вывода представлены широкой иерархией. Ключевыми классами, описывающими поведение ввода/вывода байтов, являются абстрактные классы InputStream и OutputStream. Они не реализуют конкретных механизмов обработки потоков байтов, а делегируют их своим подклассам, таким как FileInputStream/FileOutputStream.

Для понимания использования этой функциональности следует разработать приложение для анализа сигнатур произвольных файлов. Эта сигнатура позволяет определить тип содержимого файла и состоит из набора «магических чисел». Эти числа обычно располагаются в начале файла. Например, сигнатура для файла типа PNG представлена ​​первыми восемью байтами файла, которые одинаковы для всех изображений PNG:

```
89 50 4E 47 0D 0A 1A 0A
```

Вам необходимо реализовать приложение, которое принимает на вход файл signatures.txt (его следует описать самостоятельно; имя файла явно указывается в коде программы). Он содержит список типов файлов и их соответствующих сигнатур в шестнадцатеричном формате. Пример (указанный формат файла должен быть соблюден):

```
PNG, 89 50 4E 47 0D 0A 1A 0A
GIF, 47 49 46 38 37 61
```

Во время выполнения ваша программа должна принимать полные пути к файлам на диске и сохранять тип, которому соответствует сигнатура файла. Результат выполнения программы должен быть записан в файл result.txt. Если сигнатуру определить невозможно, результат выполнения будет UNDEFINED (в файл не должна быть записана никакая информация).

Пример выполнения программы:

```
$java Program
-> C:/Users/Admin/images.png
PROCESSED
-> C:/Users/Admin/Games/WoW.iso
PROCESSED
-> 42
```

Содержимое файла result.txt (загружать этот файл в качестве результата нет необходимости):

```
PNG
GIF
```

**Примечания** :

-   Мы можем точно определить тип содержимого, проанализировав сигнатуру файла, поскольку расширение файла в имени (например, image.jpg) можно изменить, просто переименовав файл.

-   Файл с подписями должен содержать не менее 10 различных форматов для анализа.

# Часть II
### Words


|||  
|--|--|  
| **Разрешенные типы**| Java Collections API, Java IO  


Помимо классов, предназначенных для обработки байтовых потоков, в Java есть классы, упрощающие обработку символьных (char) потоков. К ним относятся абстрактные классы Reader/Writer и их реализации (FileReader/FileWriter и т. д.). Особый интерес представляют классы BufferedReader/BufferedWriter, которые ускоряют обработку потоков за счет механизмов буферизации.

Теперь вам нужно разработать приложение, которое определяет степень сходства между текстами. Самый простой и очевидный способ сделать это — проанализировать частоту встречаемости одинаковых слов.

Допустим, у нас есть следующие два текста:

```
1. aaa bba bba a ccc
2. bba a a a bb xxx
```

Давайте создадим словарь, содержащий все слова из этих текстов:

```
a, aaa, bb, bba, ccc, xxx
```

Теперь создадим два вектора одинаковой длины, как и словарь. На i-й позиции каждого вектора мы отразим частоту встречаемости i-го слова в нашем словаре в первом и втором текстах:

```
A = (1, 1, 0, 2, 1, 0)
B = (3, 0, 1, 1, 0, 1)
```

Таким образом, каждый из этих векторов характеризует текст с точки зрения частоты встречаемости слов из нашего словаря. Определим сходство между векторами, используя следующую формулу:

![формула](https://markdownlivepreview.com/misc/images/formula.png)

Таким образом, значение сходства для этих векторов составляет:

```
Numerator A. B = (1 * 3 + 1 * 0 + 0 * 1 + 2 * 1 + 1 * 0 + 0 * 1) = 5
Denominator ||A|| * ||B|| = sqrt(1 * 1 + 1 * 1 + 0 * 0 + 2  * 2 + 1 * 1 + 0 * 0) * sqrt(3 * 3 + 0 * 0 + 1 * 1 + 1 * 1  + 0 * 0 + 1 * 1) = sqrt(7) * sqrt(12) = 2.64 * 3.46 = 9.1
similarity = 5 / 9.1 = 0.54
```

Ваша задача — реализовать приложение, которое принимает на вход два файла (оба файла передаются в качестве аргументов командной строки) и отображает результат их сравнения по степени сходства (косинусный метод).

Программа также должна создать файл dictionary.txt, содержащий словарь, основанный на данных из этих файлов.

Пример выполнения программы:

```
$ java Program inputA.txt inputB.txt
Similarity = 0.54
```

**Примечания** :

1.  Максимальный размер этих файлов составляет 10 МБ.
2.  Файлы могут содержать символы, отличные от букв.

# Часть III
###  File Manager

|||  
|--|--|  
| **Разрешенные типы**| Java Collections API, Java IO, Files, Paths, etc.  


Давайте реализуем утилиту для работы с файлами. Приложение должно отображать информацию о файлах, содержимом и размере папок, а также предоставлять функции перемещения/переименования файлов. По сути, приложение имитирует командную строку Unix-подобных систем.

Программа будет принимать в качестве аргумента абсолютный путь к папке, с которой мы хотим начать работу, и будет поддерживать следующие команды:

`mv`WHAT WHERE — позволяет передавать или переименовывать файл, если WHERE содержит имя файла без пути. `ls`— отображает содержимое текущей папки (имена файлов и подпапок, а также их размеры в КБ).
`cd FOLDER_NAME`— изменяет текущий каталог.

Предположим, что на диске C:/ (или в корневом каталоге, в зависимости от операционной системы) существует папка MAIN со следующей иерархией:

-   MAIN
    -   folder1
        -   image.jpg
        -   animation.gif
    -   folder2
        -   text.txt
        -   Program.java

Пример выполнения программы для каталога MAIN:

```
$ java Program --current-folder=C:/MAIN
C:/MAIN
-> ls
folder1 60 KB
folder2 90 KB
-> cd folder1
C:/MAIN/folder1
-> ls
image.jpg 10 KB
animation.gif 50 KB
-> mv image.jpg image2.jpg
-> ls
image2.jpg 10 KB
animation.gif 50 KB
-> mv animation.gif ../folder2
-> ls
image2.jpg 10 KB
-> cd ../folder2
C:/MAIN/folder2
-> ls
text.txt 10 KB
Program.java 80 KB
animation.gif 50 KB
-> exit
```
