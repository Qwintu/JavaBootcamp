# Часть I
### Яйцо, курица... или человек?

|||  
|---|---|  
| **Allowed:** |  Object, Thread, Runnable  


Курица или яйцо

Истина рождается в споре — предположим, что каждая нить даёт свой ответ. Прав тот, чья нить оставит последнее слово.

Необходимо реализовать работу двух потоков. Каждый из них должен несколько раз вывести свой ответ, например, 50 раз:

```
$ java Program --count=50
Egg
Hen
Hen
Hen
...
Egg
```

В этом случае побеждает поток egg. Однако программа также содержит основной поток. Внутри этого потока выполняется метод public static void main(String args[]). Нам нужно, чтобы этот поток отображал все свои ответы в конце выполнения программы. Таким образом, окончательный вариант выглядит следующим образом:

```
$ java Program --count=50
Egg
Hen
Hen
...
Egg
Hen
...
Human
...
...
Human
```

Это означает, что программа выводит сообщение "Human" 50 раз, которое затем печатается основным потоком.

# Часть II
### ### Яйцо, Курица, Яйцо, Курица...

|||

Keywords: | Synchronized  


Давайте организуем спор. Теперь каждая ветка обсуждения может дать свой ответ только после того, как это сделает другая ветка. Допустим, ветка, посвященная яйцу, всегда отвечает первой.

```
$ java Program --count=50
Egg
Hen
Egg
Hen
Egg
Hen
...
```

**Примечание** :  
Для решения этой задачи рекомендуется изучить принцип работы модели «производитель-потребитель».

# Часть III
### Реальная многопоточность

|||  
|---|---|  
| **Allowed:** | Object, Thread, Runnable  
Keywords: | Synchronized  


Старайтесь использовать многопоточность по прямому назначению: для распределения вычислений по всей программе.

Предположим, у вас есть массив целых чисел. Ваша задача — вычислить сумму элементов массива, используя несколько потоков суммирования. Каждый поток вычисляет сумму элементов в определенном участке массива. Количество элементов в каждом участке постоянно, за исключением последнего (его размер может изменяться в большую или меньшую сторону).

Массив генерируется случайным образом каждый раз. Длина массива и количество потоков передаются в качестве аргументов командной строки.

Чтобы убедиться в корректной работе программы, сначала следует вычислить сумму элементов массива, используя стандартный метод.

Максимальное количество элементов массива — 2 000 000. Максимальное количество потоков не превышает текущее количество элементов массива. Максимальное значение по модулю для любого элемента массива — 1000. Гарантируется корректность всех данных.

Пример выполнения программы (каждый элемент массива равен 1):

```
$ java Program --arraySize=13 --threadsCount=3
Sum: 13
Thread 1: from 0 to 4 sum is 5
Thread 2: from 5 to 9 sum is 5
Thread 3: from 10 to 12 sum is 3
Sum by threads: 13

```

**Примечания** :

-   В приведенном выше примере размер последнего раздела сводки, используемого третьим потоком, меньше, чем у остальных.
-   Потоки могут выдавать результаты операций непоследовательно.

# Часть IV
### Слишком много потоков...

|||  
|---|---|  
| **Allowed:** |  Object, Thread, Runnable  
Keywords: | Synchronized  


Допустим, нам нужно загрузить список файлов из сети. Некоторые файлы будут загружены быстрее, чем другие.

Для реализации этой функциональности мы, очевидно, можем использовать многопоточную загрузку, где каждый поток загружает определенный файл. Но что делать, когда файлов слишком много? Большое количество потоков не может работать одновременно. Поэтому многие из них будут ожидать.

Кроме того, следует помнить, что непрерывное создание и закрытие потоков — это очень ресурсоемкая операция, которой следует избегать. Гораздо целесообразнее запускать N потоков одновременно, и когда один из них завершит загрузку файла, он сможет перейти к следующему файлу в очереди.

Нам необходимо создать файл files_urls.txt (имя файла должно быть явно указано в коде программы), в котором можно указать список URL-адресов файлов для загрузки, например:

```
1 https://i.pinimg.com/originals/11/19/2e/11192eba63f6f3aa591d3263fdb66bd5.jpg
2 https://pluspng.com/img-png/balloon-hd-png-balloons-png-hd-2750.png
3 https://i.pinimg.com/originals/db/a1/62/dba162603c71cac00d3548420c52bac6.png
4 https://pngimg.com/uploads/balloon/balloon_PNG4969.png
5 http://tldp.org/LDP/intro-linux/intro-linux.pdf
```

Пример работы программы:

```
$ java Program.java --threadsCount=3
Thread-1 start download file number 1
Thread-2 start download file number 2
Thread-1 finish download file number 1
Thread-1 start download file number 3
Thread-3 start download file number 4
Thread-1 finish download file number 3
Thread-2 finish download file number 2
Thread-1 start download file number 5
Thread-3 finish download file number 4
Thread-1 finish download file number 5
```

**Примечания** :

-   Результат работы реализованной программы может отличаться от показанного на иллюстрации.
-   Каждый файл загружается только один раз одним потоком.
-   Программа может содержать «бесконечный цикл» без условия завершения (в этом случае программа может быть завершена путем прерывания процесса).
